About Debug Configuration

This dialog enables you to automatically create remote debug configurations for your applications deployed to Azure:

  1. Select the project containing the Java code that you wish to debug
  2. Specify the types of debug configurations to create (emulator and/or cloud)

If creating a cloud configuration, you need to upload your deployment to Azure first, to obtain the host address assigned to your deployment. Then enter it here.

When done, you will see the new debug configuration(s) created for your workspace under the “Remote Java Application” category.

Note: Encryption support for debugging connections is not enabled yet, therefore using remote debugging for applications deployed to the public cloud is discouraged.
